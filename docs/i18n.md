# Bilingual UI — i18n architecture (`@xidig/i18n`)

Xidig is bilingual from day one (PRD §22): **Somali-first, English fallback**.
The architecture shipped in Phase 1; Somali coverage is tracked and climbs to
100% — trust-defining surfaces (navigation, onboarding, errors, empty states,
core actions) are the launch floor and are enforced at 100% by tests.

## The one rule

**UI copy never ships as a hardcoded string.** Every user-visible string is a
key in the `@xidig/i18n` dictionaries, rendered with `t(key, params)`. This is
enforced by the `xidig-i18n/no-hardcoded-copy` ESLint rule on all `.tsx` files
under `apps/*/src` and `packages/ui/src` — hardcoded JSX text or user-facing
attributes (`alt`, `placeholder`, `title`, `aria-label`, …) fail the lint.

Legitimate escapes (brand marks, untranslatable identifiers) need an
`eslint-disable-next-line xidig-i18n/no-hardcoded-copy` with a one-line
justification comment. Endonyms in the language toggle come from
`LOCALE_NAMES`, not literals, so they don't need an escape.

## Package layout

```
packages/i18n/src/
  locales.ts          Locale type, LOCALES, DEFAULT_LOCALE ('so'), negotiateLocale()
  cookie.ts           xidig_locale cookie (name, serialize, parse)
  messages.ts         Message / PluralMessage types
  dictionaries/en.ts  English — the dictionary of record; defines every key
  dictionaries/so.ts  Somali — typed subset of the English keys
  translate.ts        createTranslator(locale) → t(key, params)
  format.ts           formatNumber / formatDate / formatRelativeTime (Intl)
  coverage.ts         coverage report + LAUNCH_FLOOR_NAMESPACES
  react.tsx           'use client' LocaleProvider, useT(), useLocale()
```

`@xidig/i18n` (the root export) is React-free — safe for server components,
API routes, scripts, and the future React Native app (API-first, §22). React
bindings are imported from `@xidig/i18n/react`.

## How to add a string

1. Add the key to `packages/i18n/src/dictionaries/en.ts` under the right
   namespace (see conventions below). English is the dictionary of record —
   a key that isn't in `en.ts` doesn't exist.
2. Add the Somali translation to `dictionaries/so.ts`. If you genuinely can't
   yet, you may ship without it — the UI falls back to English for that key —
   but keys in launch-floor namespaces **must** have Somali
   (`coverage.test.ts` fails otherwise).
3. Render it:
   - **Client component:** `const t = useT()` from `@xidig/i18n/react`.
   - **Server component:** `const t = await getT()` from `apps/web/src/lib/locale.ts`.
4. `pnpm i18n:coverage` shows the updated coverage table.

Never concatenate translated fragments — word order differs between Somali
and English. Use one key with `{param}` placeholders instead.

## Key naming conventions

- Format: `namespace.camelCaseName` (enforced by `coverage.test.ts`).
- Namespaces: `app` (identity), `nav` (canonical tabs), `term` (canonical
  product nouns), `action` (buttons/CTAs), `language`, `state`
  (loading/empty), `error` (§27 plain-language), `onboarding`, `a11y`
  (screen-reader-only), plus one namespace per screen (`home.…`).
- A new namespace must be classified in `coverage.test.ts`: launch floor
  (trust surface, Somali required) or tracked off-floor.
- Params are `{name}` placeholders; numbers are locale-formatted
  automatically.
- Messages that vary with a count are plural objects:
  `{ one: '{count} co-sign', other: '{count} co-signs' }`, selected by the
  required `params.count` (CLDR rules; English and Somali both use
  one/other).

## Language resolution & persistence

Resolution order per request (`apps/web/src/lib/locale.ts`):

1. **`xidig_locale` cookie** — set by the language toggle; 1-year Max-Age.
2. **`Accept-Language`** — quality-aware negotiation between `so` and `en`.
3. **Somali** (`DEFAULT_LOCALE`) — Somali-first product.

The toggle (`apps/web/src/components/language-toggle.tsx`) switches without a
page reload: the `LocaleProvider` re-renders client components immediately,
`router.refresh()` re-resolves server-rendered strings, and the choice is
persisted to the cookie (everyone) and to `users.preferred_language`
(signed-in members, best-effort — RLS self-row, column-granted).

## Coverage tracking

- `pnpm i18n:coverage` prints per-namespace Somali coverage and the exact
  missing keys.
- `coverage.test.ts` (runs in `pnpm test` / CI) **fails** if a launch-floor
  namespace is not 100% Somali, if a Somali key doesn't exist in English, if
  placeholders diverge between languages, or if plural shapes diverge.
- Off-floor gaps never fail the build — English fallback keeps the UI whole,
  and the report is how coverage climbs to 100% post-launch.

## Formatting helpers

`formatNumber`, `formatDate`, `formatRelativeTime` wrap `Intl` with the
active locale and fall back to English formatting if Somali CLDR data is
missing in some runtime. Use them instead of `toLocaleString()` so the locale
always comes from one place.
